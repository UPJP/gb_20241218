2. 캐시
 - cashe 자주 접근하는 데이터를 복하해놓은 임시 저장소
 - browser cache : 서버지연을 줄이기 위해 사용하는 웹 캐시의 일종

2-1. 웹캐시
 -WEB Cashe
 	-> 사용자가 (client)가 웹사이트(Server)에 접속할 때,
	    정적 컨텐츠(이미지,js,css 등) 를 특정 위치에 저장하여, 웹 사이트 서버에 해당 컨텐츠를
  	    매번 요청하여 받는 것이 아니라, 특정 위치에서 불러움으로써 사이트 응답시간을 줄이고,
	    서버 트래픽 감소효과를 볼 수 있다.

 - Browser Cache
	-> 브라우저 또는 HTTP 요청을 하는 Client
	-> 브라우저의 Back버튼 또는 이미 방문한 페이지를 재 방분하는 경우 극대화
 - Proxy Cache
	->Browser Cashe와 동일한 원리롤 동작하며, Client나 Server)가
     	   아닌 네트워크 상에서 동작
	-> 큰 회사나 IPS의 방화벽에 설치되며 대기시간을&트래픽 감소,

 -  Gateway Cashe
 	-> 서버 앞 단에 설치되어 요청에 대한 캐쉬 및 효율적인 분배를 통해
	    가용성,

4. 변수선언
 4-1. 변수의 선언
 - 선언은 var 키워드 뒤에 사용하고자 하는 변수의 이름을지정하고, 세미클론으로 한 라인 종료
 	var num;
	var username;

4-2. 변수의 할당
 - 선언된 변수에 원하는 값을 대입하는 경우

 	num =12345;
	user_ name = "홍길동"
	userName = '이순신'

4-3. 변수선언의 제약
 - 변수는 반복되어 다른 값이 할당될 수 있지만, 선언은 중복할 수 없다.
	var num =12345
       num =67890
	var num =1 		 //이미 선언된 변수이므로 불가

	my age 	19; 		//선언되지 않은 변수 사용 불가

4-4. 변수의 종류

 - javasript의 변수는 표현하는 값의 종류에 따라서 형식이 구분된다
 - 이러한 변수의 종류를 데이터형(=datatype)이라고 한다

 - Number : 숫자를 표현한다.
 - String : 문자열선언, 쌍따옴표나 홑따옴표로 감싸진 문장을 의미한다.
 - Boolen : ture, false 값을 가지는 논리 데이터 타입
 - Null : 값 없음. 개발자가 의도적으로 변수의 공간을 비워놓은 상태
 - Objact : 객체를 저장하기 위한 데이터 타입이다. 브라우저 제어 가능. form제어 ,html 태그 요소에
               대한 제어 등이 가능하다.
 - undefined : 정의되지 않음. 처음부터 변수가 선언되고 값도 할당되지 않은 상태를 의미한다. 
		    이 상태의 변수는 어떤한 처리도 불가능하다.

4-5. Null과 undefined 의 차이점
 - 프로그램 언어 java와 javascript에서 변수의 선언 차이
   java   -> 변수를 선언시에 자료형이 결정된다.
   javascript  -> 변수의 선언시에는 변수의 존재여부와 스코프(범위)만 결정되고 자료형을 값을 할당할 때 
	 	      자동으로 결정된다.

 - 할당되지 않은 값의 상태
	 java   -> 객체의 경우 null, 숫자 형태의 경우 0, boolean은 false를 기본으로 값이 할당된다
   javascript  -> 자료형이 결정되지 않은 상태이기 때문에 기본값이 할당될 수 없다. 무조건 undefined로 할당된다.
